Adding Zone Token Scopes to Your Application

Once a tenant UUID has been issued to your instace, you will need to add the zone-oath-scope to your UAA instance clients.
  1. To use the Notification service with the client credentials flow (the SDK requires this), add the zone-oath-scope to your client authorities. You can use the UAA CLI with the following command:
    uaac client update <client_name> --authorities "<list_of_authorities> notification.zone.<tenant uuid>.user"
    Tokens issued by this client will now be authorized to use the Notification service instance.
  2. To use the Notification service with a user authenticated flow (for example, predix-see), add a group with the proper scope using the UAA CLI.
    uaac client update <client_name> --scope "<list_of_scope> notification.zone.<tenant uuid>.user"

uaac group add notification.zone.<tenant uuid>.user

uaac member add notification.zone.<tenant uuid>.user <users...>
    New tokens that are issued to this user are now authorized to use the Notification service instance.
