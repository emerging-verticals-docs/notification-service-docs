Creating a Notification Service Instance

To create a service instance using a command line:
Note: The following steps are performed using the Cloud Foundry CLI tool. It is found at: https://github.com/cloudfoundry/cli. Make sure you have the latest version. To complete the steps in a web browser, follow the instructions on the service page in the Predix catalog.
  1. Use the Cloud Foundry CLI to log into Cloud Foundry. 
    cf login -a <API_Endpoint>

    The value of <API_Endpoint> is available in the Predix welcome email that you get when you register for an Predix.io user account. Depending on your Predix.io registration, the value of <API_Endpoint> is one of the following:

    • Predix Basic

      https://api.system.aws-usw02-pr.ice.predix.io

    • Predix Select

      https://api.system.asv-pr.ice.predix.io

    • Predix Japan

      https://api.system.aws-jp01-pr.ice.predix.io

    • Predix UK

      https://api.system.dc-uk01-pr.ice.predix.io

    For example,

    cf login -a https://api.system.aws-usw02-pr.ice.predix.io
  2. List the services in the Cloud Foundry marketplace:
    cf marketplace
    You see the available services and associated plans.
  3. Create a notification service instance by running the following command:
    On Mac OS and Linux, the syntax is:
    cf create-service notification <plan> <service_instance_name> -c '{“trustedIssuerId”: “<predix uaa instance url>/oauth/token”}'
    On Windows, the syntax is:
    cf create-service notification <plan> <service_instance_name> -c "{\“trustedIssuerId\”: [\“<predix uaa instance url>/oauth/token\”]}"
    where:
    • <plan> – the plan associated with a service.
    • <service_instance_name> – the service instance you are creating.
    Once a service instance is created, you are issued a tenant UUID for the created service instance. You will need to bind the application to the service instance to obtain the tenant UUID. See Binding Your Application to the Notification Service Instance.
