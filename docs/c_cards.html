Cards

A card is a component that can be shared across different environments. A card acts as a container that resides within the content area of the screen. It can contain embedded data for use when connectivity is limited or cut off. You can show and hide cards as needed.

A card can receive and transmit events, and is both aware of and reactive to the context in which it resides. Components can interact within an individual card, and a card can interact with other cards. You can also create a deck from a collection of cards to create more intricate user interactions. See Customizing Web Applications with Predix UI for information about how cards and decks are used.

You can save the complete context of a card using a View service APII.

For more information about this API, see the API Documentation.
