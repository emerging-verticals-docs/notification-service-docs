Creating a UAA Service Instance for the Predix UI Seed

  1. Sign into your Predix account at https://www.predix.io.
  2. Navigate to Catalog > Services, then click the User Account and Authentication tile.
  3. Click Subscribe on the required plan.
  4. Complete the fields on the New Service Instance page.
    Field Description
    Org Select your organization.
    Space Select the space for your Predix UI Seed web application.
    Service instance name Enter a unique name <uaa_instance> for this UAA service instance .
    Service plan Select a plan.
    Administrative client secret Enter an administrative client secret <admin_client_secret> that serves as the password for this UAA service instance. The admin_client_secret can be any alphanumeric string.
    Note: Record the admin_client_secret in a secure place for later use.
    Subdomain (Optional) Enter a subdomain you might need to use in addition to the domain created for UAA. Do not use special characters in the name of the subdomain. The value of sub-domain is case insensitive.
  5. Click Create Service.

Your UAA service instance <uaa_instance> is created with the following specification:

  • Administrative client admin.
    Note: The administrative client admin is created for bootstrap purposes. You can create additional clients to use with your Predix UI Seed web application. For more information, see Creating an OAuth2 Client.
  • Administrative client secret <admin_client_secret> that you specified while creating the UAA service instance.
