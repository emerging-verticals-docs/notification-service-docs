View Service Database Operations

View service API operations on card and deck objects

You store the state of card or deck objects in the View service database. You can also create the following relationships between card and deck objects in the View service database:
  • Associate a card with a deck.
  • Tag a card or deck to create categories.
Note: See the StrongLoop documentation for information about additional data operations such as ordering and filtering (https://docs.strongloop.com/display/public/LB/LoopBack).

Card CRUD Operations

The following View service API supports operations on card objects.
HTTP Method Path Description
GET /api/cards Get all cards.
POST /api/cards Create cards.
PUT /api/cards/Id Update card Id.
DELETE /api/cards/Id Delete card Id.

For more information about this API, see the API Documentation.

See Creating a Card for an example of creating a card instance object in the View service database

Deck CRUD Operations

The following View service API supports operations on deck objects.
HTTP Method Path Description
GET /api/decks Get all decks.
POST /api/decks Create decks.
PUT /api/decks/Id Update specified deck.
DELETE /api/decks/Id Delete specified deck.

For more information about this API, see the API Documentation.

See Creating a Deck for an example of creating a deck instance object in the View service database.

Associating Cards and Decks

The following View service API associates card and deck objects.
HTTP Method Path Description
POST /api/decks/Id/cards/add Associate a card Id with a deck.
POST /api/decks/Id/cards/remove Detach a card Id from a deck.
GET /api/decks/Id/cards Get all cards in a deck.
POST /api/decks/Id/cards/ordered Set the card order in a deck.

For more information about this API, see the API Documentation.

See Associating a Card with a Deck for an example association between a card and a deck in the View service database.

Tagging Cards and Decks

The following View service API tags card or deck objects to create categories.
HTTP Method Path Description
POST /api/cards/Id/tag Tag a card.
POST /api/decks/Id/tag Tag a deck.
GET /api/cards/tag?values={comma delimited tag strings} Get all cards with tag strings.
GET /api/decks/tag?values={comma delimited tag strings} Get all decks with tag strings.

For more information about this API, see the API Documentation.

See Tagging a Deck for an example of deck tagging in the View service database.
