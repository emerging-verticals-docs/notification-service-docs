Using View Service

View service tasks

Select a REST client (such as Postman) that can execute API requests to use View service.

For more information about these APIs, see the API Documentation.

View service stores the state and layout of card and deck objects in a database. The View service database also stores relationship information between card and deck objects.

As a developer, you can use the View service database to share and recreate saved cards, as well as send the context of a deck object (including cards and associated components) to render as a view in a web browser. This deck-based modular view framework gives an application user the flexibility to select a specific view at run time to grasp information intuitively and make time-critical decisions.

Follow these steps to manage card and deck objects using View service:

  1. Create a card in the View service database that can be retrieved in JSON format.
  2. Create a deck in the View service database that can be retrieved in JSON format.
  3. Associate one or more cards with a deck in the View service database and retrieve all cards.
  4. Tag a deck or card in the View service database to look up by category.
  5. Query data in the View service database with filters and return the results in JSON format.

See Displaying a Multi-Card Deck for an example web application that displays cards in a deck using View service to control the visual summary of user information.
