Query Data with Filters

Overview of using a REST client to query data with filters

A query is a read operation on the View service database that returns a result set in JSON format. Filters specify the critera for the result set. Select a REST client (such as Postman) that can execute API requests to query data wth filters.

You specify a filter on the View service database by using the following format:

?filter[<filterType>]=[<filter inputs>]
Table 1. Supported Filters
Filter Type Description Example
include Allow related data models to be included in the result set /decks?filter[include]=cards
order Allow result set to be return in order that is specified /decks?filter[order]=createTimeStamp DESC
  Get a deck with ordered cards decks?filter[cards] &filter[order]=cards
where Filter the result set by field value /decks?filter[where][title]=Overview
Note: See the StrongLoop documentation for more infromation about querying data with filters (https://docs.strongloop.com/display/public/LB/Querying+data).
