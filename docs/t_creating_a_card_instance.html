Creating a Card

Procedure for creating a card instance in the View service database.

Select a REST client (such as Postman) that can execute API requests to use View service. .

For more information about this API, see the API Documentation.

A card is a component that provides a composable user interface object. If you have access to the public Github repository, consider cloning or forking one of the sample cards in the px-sample-cards repository rather than building one from scratch.

  1. Create a card instance in the View service database. For example:
    1. Add a card to the <body> section of the HTML page of your web application. For example:
      <!-- a card that can display temperature of a city -->
<my-temperature-card id="card-1" title="San Ramon Temperature"postal="94582" unit="F"></my-temperature-card>
    2. In a REST client, enter the following POST request:
      [POST] /api/cards
    3. Copy the following JSON blocks into the request body:
      [{
    "title": "San Ramon Temperature",
    "slug": "my-temperature-card",
    "attributes": {
        "postal": "94582",
        "unit": "F"
    }
}]
  2. In a REST client, retrieve a card instance in JSON format from the database. For example:
    1. In a REST client, enter the following GET request:
      [GET] /api/cards
    2. Copy the following JSON blocks into the request body:
      [{
    "title": "San Ramon Temperature",
    "slug": "my-temperature-card",
    "attributes": {
        "postal": "94582",
        "unit": "F"
    }
}]
