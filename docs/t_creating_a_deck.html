Creating a Deck

Procedure for creating a deck instance in the Views service database.

Select a REST client (such as Postman) that can execute API requests to use Views service.

For more information about this API, see the API Documentation.

A deck is a container that determines the layout of cards.
  1. Create a deck instance in the Views service database. For example:
    1. Add a deck to the <body> section of the HTML page of your web application:
      <!-- a deck with 2 cards -->
<px-deck id="deck-1">
    <my-temperature-card id="card-1" title="San Ramon Temperature"postal="94582" unit="F"></my-temperature-card>
    <my-temperature-card id="card-3" title="Walnut Creek Temperature" postal="94595" unit="F"></my-temperature-card>
</px-deck>
    2. In a REST client, enter the following POST request:
      [POST] /api/decks
    3. Copy the following JSON blocks into the request body:
      {
    "title": "Contra Costa Deck"
    "cardOrder": [{
                   "title": "San Ramon Temperature",
                   "slug": "my-temperature-card",
                   "attributes": {
                       "postal": "94582",
                       "unit": "F"
                     }     
                   },
                   {
                    "title": "Walnut Creek Temperature",
                    "slug": "my-temperature-card",
                    "attributes": {
                        "postal": "94595",
                        "unit": "F"
                     }
                  }]
    "id": "deck-1"
}
  2. In a REST client, retrieve a deck instance in JSON format from the database. For example:
    1. Enter the following GET request:
      [GET] /api/decks
    2. Copy the following JSON blocks into the request body:
      {
    "title": "Contra Costa Deck"
}
