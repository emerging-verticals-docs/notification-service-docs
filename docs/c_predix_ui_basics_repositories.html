Predix UI Basics Repositories

The Predix UI basics repositories serve as building blocks to create the user interfaces of a web application.

Predix UI includes a set of basics repositories in the PredixDev public GitHub organization. The Predix UI basics repositories are used in isolation or combined to create more complex Predix UI components.

As a developer, you can use the code in the following Predix UI basics repositories to produce these interactions.

Table 1. Predix UI Basics Repositories
Repository Description Documentation
px-alert-label Creates default alert messages. API Reference
px-file-upload Component that provides users the ability to upload one or more files, whether as part of a larger form, or as a standalone component. API Reference
px-inbox Displays a list of items with detailed information or actions associated with each item. API Reference
px-popover Displays additional information to a user. API Reference
px-progress-bar Displays a progress bar that indicates the progress of an activity with explicit start and end times, or known duration. API Reference
px-spinner Indicates when data is being loaded, a view is changing, or any place a visual indication is needed to show that work is being performed for an indefinite period of time. API Reference
px-tabs Creates a set of tabs to insert content. API Reference
px-tooltip Displays additional information to users once hovered over the target element for a specified amount of time. API Reference
px-typeahead Searches user input and lists all the suggestions which has the input API Reference
px-validation Element that provides built-in as well as developer-supplied validation solutions. API Reference
Note:

The API reference documentation for individual repositories also includes usage instructions and examples.

File a GitHub issue if you encounter a problem using a Predix UI repository. For more information on filing a GitHub issue, see https://guides.github.com/features/issues/.
