View Service Database Setup

You can use the View service database to store the state and layout of card and deck instances, as well as the relationship information between card and deck instances.

Before you begin:

For more information about the View service database, see:
  • About View Service
  • View Service Database Operations

Select a REST client (such as Postman) that can execute Vview service API requests.

For more information about these APIs, see the API Documentation.

Task Roadmap

Follow these tasks to set up the View service database:

# Task Information
1 Create a card instance in the View service database that can be retrieved in JSON format. See Creating a Card
2 Create a deck instance in the View service database that can be retrieved in JSON format. See Creating a Deck. .
3 Associate one or more card instances with a deck instance in the View service database and retrieve all card instances. See Associating a Card with a Deck.
4 Tag a deck or card instance in the View service database to look up by category. See Tagging a Deck.
5 Query data in the View service database with filters and return the results in JSON format. See Query Data with Filters.

What to do next:

See Displaying a Multi-Card Deck for an example web application that displays cards in a deck using the View service database to control the visual summary of user information.
