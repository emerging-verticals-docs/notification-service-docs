Troubleshooting Predix UI Seed Setup

You may encounter these issues that when setting up the Predix UI Seed.

Permission denied to access your GitHub account

You may receive the mesage Error: Permission denied when you try to log into your GitHub account.

Cause

Your connection was refused by the server.

Solution

You may need to generate and add a public SSH key to your GitHub account.

See Creating the Predix Developer Environment for more information.

No access to the PredixDev organization in GitHub

You cannot access the Predix UI repositories in the PredixDev organization after you log into your GitHub account.

Cause

You may not have registered for a Predix platform user account.

Solution

You may need to register for a Predix Platform account. See Registering for a Predix Platform User Account for more information.

Errors installing the Node.js npm package manager

You may receive the messaage EACCESS when you try to install a Node.js npm package globally.

Cause

You do not have permission to write to the directories that the Node.js npm package manager uses to store global packages and commands.

Solution

You can fix this issue using one of the following options:

  • Change the file permissions to the default directory for the Node.js npm package manager.
  • Change the default directory for the Node.js npm package manager to another directory.

See https://docs.npmjs.com/getting-started/fixing-npm-permissions for more information.

Access to remote resources blocked through a proxy connection

Access to certain tools (such as cURL, Maven, Git, or Eclipse) is blocked.

Cause

If traffic between your corporate network and the Internet is monitored, access to remote resources may be blocked by the proxy.

Solution

You may need to change your proxy settings to access remote resources.

  • See Defining Proxy Connections to Remote Resources for more information about changing proxy settings to access remote resources. Contact your IT administrator for the correct proxy settings.
  • After you have installed the latest version of the Git client for your operating system, you need to configure the ~./gitconfig file to access remote resources through a proxy connection. 
    [url "https://"]
    insteadOf = git://
